# Project Guidelines for Livestock and Heat MERRA-2 Analysis

This file contains project-specific guidelines and best practices for working with MERRA-2 data in this repository.

## Data Visualization Standards

### MERRA-2 Gridded Data Plotting

When creating visualizations of MERRA-2 gridded data (temperature, VPD, soil moisture, precipitation, etc.), **always use Method 5: Grid Edges + Boundaries**.

#### Why This Approach?

MERRA-2 data is gridded climate reanalysis at ~55km resolution (~0.5° × 0.625°). The data represents discrete grid cells, not continuous fields. Each cell is assigned to a state/region based on its centroid location (point-in-polygon method).

**The Problem:**
- Grid cells are large (~55km)
- Cells near state boundaries may have centroids in one state but overlap another
- ~38% of US grid cells are on state boundaries
- Vector state boundaries don't align perfectly with gridded data

**The Solution:**
- **Show grid cell edges** to make discrete resolution clear
- **Include state boundaries** for geographic context
- Makes it obvious this is ~55km gridded data, not continuous
- Prevents confusion about boundary mismatches

#### Implementation

All MERRA-2 plots should use the standardized configuration from `plot_config.py`:

```python
# Import the config module
import plot_config as pc

# Method 1: Use helper functions (recommended)
im, ax = pc.plot_merra2_variable(t2m_data, 'T2M', lons, lats, title='Daily Mean Temperature')
plt.show()

# Method 2: Manual plotting with standard parameters
fig, ax = plt.subplots(figsize=(12, 8), subplot_kw={'projection': ccrs.PlateCarree()})
ax = pc.setup_map_axis(ax)

style = pc.get_variable_style('T2M')
im = ax.pcolormesh(
    lons, lats, t2m_data,
    cmap=style['cmap'],
    vmin=style['vmin'],
    vmax=style['vmax'],
    edgecolors='white',      # Show grid cell boundaries
    linewidth=0.2,           # Thin lines
    alpha=0.95,              # Slight transparency
    transform=ccrs.PlateCarree()
)
plt.colorbar(im, ax=ax, label=style['label'])
plt.show()
```

#### Standard Grid Parameters

Add these parameters to **every** `pcolormesh()` call for MERRA-2 data:

```python
edgecolors='white'     # Grid cell edges
linewidth=0.2          # Thin but visible
alpha=0.95             # Slight transparency to see edges better
```

#### Standard Map Features

```python
# State boundaries
ax.add_feature(cfeature.STATES, linewidth=0.5, edgecolor='black')

# Coastline
ax.add_feature(cfeature.COASTLINE, linewidth=0.8)

# Extent (US Lower 48)
ax.set_extent([-125, -66, 24, 49], crs=ccrs.PlateCarree())

# Gridlines
gl = ax.gridlines(draw_labels=True, linewidth=0.5, alpha=0.5, linestyle='--')
gl.top_labels = False
gl.right_labels = False
```

## Region and State Masks

### Assignment Method

Grid cells are assigned to states/regions using **point-in-polygon with centroids**:
- If the grid cell center point falls within a state boundary, the entire cell is assigned to that state
- This is the standard method for climate data analysis
- Alternative methods (area-weighted, rasterization) are more complex and not necessary for this analysis

### Region Definitions

The project uses 10 official US regions:

1. **Region 1**: CT, ME, NH, VT, MA, RI
2. **Region 2**: NY, NJ
3. **Region 3**: DE, MD, PA, WV, VA
4. **Region 4**: AL, FL, GA, KY, MS, NC, SC, TN
5. **Region 5**: IL, IN, MI, MN, OH, WI
6. **Region 6**: AR, LA, NM, OK, TX
7. **Region 7**: IA, KS, MO, NE
8. **Region 8**: CO, MT, ND, SD, UT, WY
9. **Region 9**: AZ, CA, HI, NV
10. **Region 10**: AK, ID, OR, WA

**Note**: Alaska (AK) and Hawaii (HI) are in the mapping but not in the current grid bounds (US Lower 48: 24-49°N, 125-66°W).

### Using Masks

```python
# Load mask (use config for paths)
import config

# Filter by region
data_region = data.where(mask_ds.region_mask == region_id)

```

## MERRA-2 Data Processing

### Collections Used

- **M2T1NXSLV**: Hourly single-level diagnostics (T2M, VPD)
- **M2T1NXLND**: Hourly land surface diagnostics (soil moisture, precipitation)

### Processing Tools

Use **CDO (Climate Data Operators)** for data processing, not pure Python/xarray:
- 5-10x faster than xarray
- Handles spatial subsetting, unit conversions, and compression efficiently
- See `process_merra2_cdo_t2m_vpd.ipynb` for temperature/VPD
- See `process_merra2_soil_precip_cdo.ipynb` for soil/precipitation

### Output Format

- **Format**: NetCDF4 Classic
- **Compression**: Deflate level 4
- **Precision**: float32 with 5 significant digits
- **Temporal**: Hourly (24 timesteps per day)
- **Spatial**: US Lower 48 (24-49°N, 125-66°W)

## Source Modules (src/)

### Purpose

All reusable functions are extracted to `src/` modules to:
- **Avoid code duplication** across notebooks
- **Ensure consistency** in analysis methods
- **Improve maintainability** with single source of truth
- **Enable testing** of core functions

### Available Modules

#### src/data_loading.py
Functions for loading NetCDF and CSV data:
- `load_monthly_data()` - Load processed monthly NetCDF files
- `load_region_mask()` - Load state/region masks
- `create_cattle_region_masks()` - Create boolean masks for cattle regions
- `load_cattle_data()` - Load USDA cattle slaughter CSV data

#### src/aggregation.py
Spatial and temporal aggregation:
- `aggregate_to_region()` - Compute spatial mean for a region
- `aggregate_to_weekly()` - Aggregate daily data to weekly (Saturday ending)

#### src/thermal_acclimation.py
Thermal Acclimation State (TAS) calculations:
- `calculate_tas()` - Calculate TAS from daily metrics
- `calculate_tas_statistics()` - Summary statistics for TAS

#### src/weekly_operations.py
Weekly data operations:
- `create_lagged_variables()` - Create 1-4 week lagged predictors
- `reshape_cattle_to_long()` - Reshape cattle data wide → long
- `merge_cattle_climate()` - Merge climate predictors with cattle outcomes

#### src/utils.py
Utility functions:
- `generate_year_months()` - Generate (year, month) tuples for date range
- `get_lat_lon_coords()` - Extract lat/lon from NetCDF files
- `check_file_exists()` - Check if output file exists
- `validate_date_range()` - Validate date range parameters
- `format_file_size()` - Human-readable file sizes

#### src/processing.py
MERRA-2 threshold calculations:
- `load_us_mask()` - Load US state mask
- `compute_daily_daytime_stats()` - Daytime heat threshold hours
- `compute_daily_nighttime_stats()` - Nighttime recovery hours
- `compute_daily_vpd_stats()` - Afternoon VPD statistics
- `process_month()` - Generic month processing function

#### src/analysis.py
Common analysis functions for notebooks:
- `load_weekly_data()` - Load weekly aggregated datasets
- `compute_state_mean()` - Compute spatial mean for a state
- `compute_state_stats()` - Multiple statistics (mean, std, min, max, median)
- `mann_kendall_test()` - Non-parametric trend test
- `get_season()` - Get season name from month number
- `resample_to_monthly()` - Resample weekly → monthly data
- `normalize()` - Normalize data (zscore, minmax, or robust)
- `identify_heatwaves()` - Identify heatwave events in time series

### Using src/ Modules in Notebooks

**Always import from src/ modules instead of defining functions inline:**

```python
# At the top of your notebook
import sys
from pathlib import Path

# Add project root to path
sys.path.append(str(Path('../..')))  # Adjust based on notebook location

# Import from config and src
import config
from src.data_loading import load_monthly_data, load_cattle_data
from src.aggregation import aggregate_to_region, aggregate_to_weekly
from src.analysis import compute_state_mean, mann_kendall_test
```

**Benefits:**
- **60-80% code reduction** in notebooks
- **Consistency** across all analyses
- **Easy updates** - fix once in src/, applies everywhere
- **Better testing** - functions can be unit tested
- **Documentation** - NumPy-style docstrings with examples

- **Examples section** with usage demonstrations

## Code Style

### Byte vs String Handling

NetCDF files store state abbreviations as bytes (char type). Always decode when displaying:

```python
# Reading from NetCDF
state_abbr = mask_ds.state_abbr.values[i]
state_abbr_str = state_abbr.decode() if isinstance(state_abbr, bytes) else state_abbr

# Comparing/searching
state_abbr_bytes = 'TX'.encode() if isinstance('TX', str) else 'TX'
state_idx = np.where(mask_ds.state_abbr.values == state_abbr_bytes)[0]
```

### Notebook Organization

All notebooks should follow this structure:

1. **Imports and Setup**
   ```python
   import sys
   from pathlib import Path
   sys.path.append(str(Path('../..')))

   import config
   from src.module import function1, function2
   ```

2. **Configuration**
   - Use `config.py` for all paths
   - Define analysis-specific parameters
   - Create output directories using `config.FIGURES_DIR`

3. **Data Loading**
   - Use src/ functions (e.g., `load_weekly_data()`, `load_cattle_data()`)
   - Load masks using `config.MASK_FILE`

4. **Analysis**
   - Use src/ functions for common operations
   - Document analysis steps with markdown cells
   - Save intermediate results if needed

5. **Visualization**
   - Use standardized plotting configuration
   - Save figures to `config.FIGURES_DIR / 'analysis_type'`
   - Include clear titles and labels

6. **Cleanup**
   - Close datasets to free memory
   - Print summary statistics or completion message
